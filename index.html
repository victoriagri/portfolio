Project excerpt

LHannun’s login flow had major navigation issues, especially on mobile screens. My focus was to ensure users could complete this step successfully without interruptions caused by UI or structural problems.

Problem

Many users experienced navigation issues during the login process, particularly on mobile devices.This friction often caused confusion, abandonment, and increased support requests.

My role

I was responsible for improving the login experience by redesigning the user flow and interface, ensuring it was fully responsive and intuitive on mobile.

Responsibilities

  • Research
  • Low- and high-fidelity prototyping
  • Design system iteration
  • A/B testing

Solution

I proposed a new structure for the login user flow, redefining the information architecture and visual hierarchy to make the experience clearer and more intuitive.

By mapping the user flow and restructuring the architecture, I was able to establish clearer hierarchies and reduce unnecessary steps.

I then created low-fidelity wireframes to align with developers early on, followed by high-fidelity designs consistent with the existing design system.

Validation

The new login flow was tested through A/B testing to compare performance against the previous version.

The results were positive: the updated design reduced user abandonment and improved overall usability.

Impact

  • One month after implementation, customer service complaints related to login, purchases, and website structure decreased by 75%.
  • Based on these results, the login experience continues to be iterated to further improve navigation and usability.

Scaling Conversion & Systemizing UX at Hannun

Product Designer

My role

Product Designer & Design Owner

Team

2 Developers + Technology Team

Scope

Funnel optimization
Login redesign
Information architecture
Design system creation
A/B testing & iteration

Year

2021–2022

Context

Hannun’s ecommerce platform was experiencing friction in the login and account flow, particularly on mobile. This interruption affected the checkout journey, increased abandonment, and generated a high volume of customer service complaints.

As the design owner, I led the redesign of the login funnel while progressively building a scalable design system to improve consistency and long-term product growth.

Problem

The login and account architecture disrupted the purchase flow. Users struggled with navigation clarity, hierarchy, and mobile usability. This resulted in:

  • Checkout abandonment
  • Confusion in account management
  • High support ticket volume

Approach

Rather than executing a full redesign, I proposed a progressive improvement strategy to reduce friction without disrupting active users.

  • Redefined the login and account user flow
  • Simplified architecture and action hierarchy
  • Prioritized mobile-first usability
  • Aligned decisions with development feasibility

Systemization

To sustain improvements and scale efficiently, I created and iterated a structured UI Kit and design guidelines. This ensured consistency across components, typography, spacing, and interaction patterns.

The system reduced design ambiguity, accelerated collaboration with developers, and enabled faster experimentation.

Validation

The new login flow was tested through A/B testing to validate performance against the previous version. The results confirmed a reduction in friction and abandonment.

Impact

  • Customer service complaints related to login and account issues decreased by 75% within the first month.
  • Conversion rate reached 0.41% in 2022.
  • Average basket size increased from €143 to over €160 (+11%).
  • The system enabled continuous iteration without large disruptive redesigns.

Project excerpt

Redesign of a global design system for a multinational laboratory working across multiple brands. The project focused on creating a scalable and consistent foundation through design tokens, core components, and structured documentation in Figma, improving alignment between design and development teams while ensuring long-term system maintainability.

Problem

The existing design system lacked a solid and scalable foundation. Styles were not tokenized, components were inconsistent, and documentation was limited, making it difficult to maintain coherence across multiple brands and slowing down both design and development workflows.

Responsibilities

  • Audit of the existing design system
  • Definition of global structure and component architecture
  • Creation of design tokens and variables
  • Design and organization of core components
  • Documentation of usage, layout, and styling in Figma
  • Collaboration with stakeholders and development team during validation and handoff

Solution

A scalable global design system was defined to unify design across multiple brands.

The solution focused on establishing a strong foundation through design tokens, a consistent component architecture, and centralized documentation in Figma, enabling more efficient collaboration between design and development teams.

Impact

  • The new design system established a solid and scalable foundation across multiple brands. It improved visual and functional consistency, reduced duplicated work, and enabled design and development teams to work more efficiently with a shared system and clear documentation.

Variable Creator — Figma Plugin

UX / UI · Product Design

Project excerpt

Variable Creator is a Figma plugin designed to streamline the creation and management of design tokens. It helps designers build primitives, aliases, and scalable systems using native Figma variables.

Problem

While working extensively with design systems, I noticed that creating and managing variables in Figma was still a slow and repetitive process.

Existing tools didn’t fully automate the most mechanical parts of the workflow, forcing designers to spend time on setup instead of decision-making.

My role

Product designer responsible for defining the problem, designing the user experience, and crafting the interface of the plugin from concept to launch.

Responsibilities

  • Problem definition based on real design system workflows
  • User experience and interaction design for a technical tool
  • High-fidelity UI design for a Figma plugin environment
  • Designing scalable flows for primitives, aliases, and tokens
  • Iterative testing and refinement during development

Description

Variable Creator was built as a response to daily friction encountered while working with complex design systems. The goal was to reduce cognitive load and repetitive tasks when creating variables.

The plugin focuses on speed, clarity, and flexibility, allowing designers to generate color scales, aliases, spacing, radius, and typography tokens using native Figma variables.

By automating setup tasks, the tool enables designers to spend more time on decisions that require judgment, consistency, and system thinking.
